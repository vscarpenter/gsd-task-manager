3. Security & Supply Chain

High CSP allows 'unsafe-inline' for script-src in production

File/Line: app/layout.tsx:21

Issue: script-src 'self' 'unsafe-inline' allows injected inline scripts to execute, weakening XSS protection.

Confidence: High.

Recommendation: Replace with nonce-based CSP. Next.js supports nonce via headers() in middleware. Alternatively, use 'strict-dynamic' with a script loader.

Effort: Medium (requires testing all inline scripts including theme-toggle)

Medium connect-src allows all HTTPS — overly permissive

File/Line: app/layout.tsx:33 — connect-src 'self' https: wss:

Issue: Any HTTPS endpoint can be contacted. Should be scoped to https://api.vinny.io and wss://api.vinny.io.

Confidence: High.

Recommendation: connect-src 'self' https://api.vinny.io wss://api.vinny.io https://*.ingest.sentry.io

7. Performance

Low Batch sync lookups — fetchRemoteTaskIndex pre-fetches all IDs

The sync engine avoids N+1 by fetching all remote task IDs in one request (fetchRemoteTaskIndex()) rather than individual lookups per task. Push operations are throttled at 100ms between requests to avoid PocketBase 429 errors.

Low Virtual scrolling for large task lists

@tanstack/react-virtual is used for rendering only visible DOM rows, preventing degradation with hundreds of tasks.
