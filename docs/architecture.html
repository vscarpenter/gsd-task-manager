Architecture

GSD Task Manager

Next.js 16 PWA Offline-First IndexedDB PocketBase Sync

A privacy-first Eisenhower matrix task manager. All data lives locally in IndexedDB; cloud sync is opt-in to a self-hosted PocketBase instance. Built as a static-export PWA deployed to S3/CloudFront.

TL;DR
  1. All data lives in IndexedDB via Dexie (schema v14, 10 tables). The app is fully functional offline with zero infrastructure.
  2. Cloud sync is opt-in via self-hosted PocketBase. Last-write-wins conflict resolution with client_updated_at timestamps. SSE realtime for instant cross-device updates.
  3. Static export to S3/CloudFront with a three-cache service worker strategy. No SSR, no API routes, no hydration complexity.
  4. MCP server provides 20 AI tools for Claude Desktop integration. All writes support dry-run mode.
  5. Key risks: LWW data loss on concurrent edits, single EC2 instance for PocketBase, no automated backups, documentation drift from reality.

Overview

GSD organizes tasks into an Eisenhower matrix using two boolean flags: urgent and important. The v9 architecture replaced a multi-view layout with a single-matrix shell, capture bar, and edit drawer.

Q1 — Do First

Urgent + Important. Crisis, deadlines, emergencies.

Q2 — Schedule

Not Urgent + Important. Planning, growth, prevention.

Q3 — Delegate

Urgent + Not Important. Interruptions, some meetings.

Q4 — Eliminate

Not Urgent + Not Important. Time wasters, busy work.

User Experience

The v9 architecture centers on a single-matrix view with progressive disclosure: a capture bar for fast entry, an edit drawer for detail, and keyboard shortcuts for power users.

Interaction Model

Capture Bar

Press n to focus. Type with ! (urgent), * (important), #tag markers. Tab cycles quadrant. Enter to submit. Shift+N escalates to full drawer.

Edit Drawer

Slide-in panel over the matrix preserving spatial context. Supports title, description, quadrant picker, due-date presets, tags, subtasks, and dependencies.

Drag & Drop

@dnd-kit with 8px activation distance. Tasks dragged between quadrants with ghost overlay. Drop targets highlight with quadrant accent color.

Command Palette

Cmd+K opens cmdk-based palette for quick actions and task search. Mounted in AppShell on every route.

Navigation Architecture

Every route shares the AppShell component providing a collapsible icon rail (desktop) or bottom tab bar (mobile), topbar with search, help drawer, and command palette.

Matrix → Dashboard → Settings → Archive → About

Route groups: (matrix), (dashboard), (archive), (sync), (pwa), plus standalone /settings and /about

Accessibility

What works well
ARIA landmarks on navigation, aria-pressed on toggles, aria-current="page" on active nav, Radix Dialog with focus trapping for help drawer, focus-visible rings, 44x44px mobile touch targets, prefers-reduced-motion support.
Gap: Edit Drawer accessibility
The edit drawer is a hand-rolled <div role="presentation"> overlay with no aria-modal, no focus trap, and a fragile setTimeout for initial focus. Unlike the help drawer (Radix Dialog), keyboard-only and screen-reader users may experience focus escaping behind the overlay. components/matrix-simplified/edit-drawer.tsx
Gap: Mobile discoverability
Search is hidden sm:block on mobile. The drag grip icon is opacity-0 until hover (invisible on touch). Sync entry point is also hidden on mobile. Three features invisible to a mobile-only user.

Technical Architecture

GSD is a fully client-side application. All components use "use client" — there is no server rendering. The static export eliminates hydration mismatches, SSR infrastructure, and server/client divergence.

Module Structure

lib/

40+ files. Data layer core: db.ts (Dexie schema), tasks.ts (CRUD), schema.ts (Zod), use-tasks.ts (live queries), quadrants.ts (Eisenhower logic). Subdirectories: analytics/, notifications/, sync/.

lib/sync/

18 files. Modular sync system: pocketbase-client.ts, pb-sync-engine.ts (push/pull), pb-realtime.ts (SSE), pb-auth.ts (OAuth), task-mapper.ts, sync-coordinator.ts, health-monitor.ts, queue.ts, retry-manager.ts.

components/

Domain-specific groups: matrix-simplified/ (v9 shell), task-card/ (composable card), sync/, settings/, settings-page/, dashboard/, command-palette/.

packages/mcp-server/

Standalone workspace. 7 read tools, 5 write tools (with dry-run), 5 analytics tools, 3 system tools. Stdio transport, TTL cache, retry logic.

Data Flow

Write path:

User Action → Zod Validation → Dexie Transaction → IndexedDB → PocketBase Push (async, optional)

Read path:

IndexedDB → useLiveQuery → React Component → Auto Re-render

Sync path:

sync-coordinator → pb-push (100ms throttle) ↔ PocketBase API ↔ pb-pull (LWW) → Dexie

Persistence & Sync

All data lives locally in IndexedDB via Dexie. Cloud sync is an opt-in layer that pushes and pulls to a self-hosted PocketBase instance using last-write-wins conflict resolution.

Task Data Model

Tasks are validated by Zod schemas at all data boundaries. The quadrant is derived from urgent + important booleans via resolveQuadrantId().

Core Fields

id, title, description, urgent, important, quadrant, completed, completedAt, dueDate, createdAt, updatedAt

Nested Structures

subtasks[] (id/title/completed), tags[], dependencies[] (task IDs), timeEntries[] (start/end/notes)

Recurrence & Snooze

recurrence pattern, snoozedUntil. Completed recurring tasks auto-create the next instance.

Sync Protocol

Last-Write-Wins (LWW)
Conflict resolution uses client_updated_at timestamps. Remote wins if its timestamp is newer. A 30-second overlap window on pull cursors and a 5-minute future clamp handle boundary conditions and clock drift.
OperationMechanismSafeguards
PushDrain syncQueue sequentially, 100ms throttlePre-fetch remote index, LWW guard before write, abort on HTTP 429
PullFetch by owner + client_updated_at >= cursor30s overlap window, preserve device-local fields, skip invalid records
RealtimePocketBase SSE subscribe('*')Echo filtering via device_id, owner validation, Zod schema check
Delete ReconciliationCompare remote index vs local after pullProtected by pending-queue check and fetchSucceeded guard
Field MappingcamelCase (local) ↔ snake_case (PocketBase)Bidirectional Zod validation with .strip()

Import / Export

Export

Strict validation — taskRecordSchema.strict() rejects unknown fields. Wraps in {tasks, exportedAt, version} payload. Skips corrupt tasks with warning.

Import

Lenient validation — .strip() silently drops unknown fields (e.g., legacy vectorClock). Two modes: replace (clear + replace) or merge (regenerate conflicting IDs). Safety limits: 10K tasks, 10MB max.

Risk Assessment

A constructively critical review of the architecture, surfacing contradictions, complexity hotspots, and hard questions that need honest answers.

Documentation vs. Reality

ADR-0011 documents deletions that never happened
The ADR states share-task-dialog/, lib/confetti.ts, and canvas-confetti should be deleted. All still exist and are actively imported in components/matrix-simplified/index.tsx. This ADR describes aspirations, not reality.
CLAUDE.md says the command palette is "not wired in"
app-shell.tsx imports and renders <CommandPalette> on every page. The documentation is stale. The help drawer’s keyboard shortcuts section doesn't mention Cmd+K. Users may not discover the feature; developers may not trust CLAUDE.md.

Single Points of Failure

RiskImpactMitigation
PocketBase on single EC2 All synced users lose multi-device capability None documented. No failover, no multi-region.
No automated PocketBase backup Disk failure loses all synced data permanently SQLite on EC2 with no backup scripts or S3 snapshots.
Hand-written service worker A bug silently breaks offline for all users TypeScript source exists but must be manually synced to JS copy.
LWW data loss Simultaneous edits to different fields silently loses one set Acknowledged in ADR-0003. No conflict visibility for users.

Security Surface

Auth tokens in localStorage
PocketBase auth tokens live in localStorage via the SDK’s authStore. Any XSS vulnerability grants full read/write access to all synced task data. No encryption in the sync layer.
MCP server bearer token
GSD_AUTH_TOKEN is a long-lived bearer token visible in shell history, process listings, and Claude Desktop config files. Token rotation is not documented.

Dead Weight

Coverage-Gaming Tests

14 test files named coverage-boost.test.ts, final-coverage-push.test.ts, gap-closing-2.test.tsx. These exist to hit the 80% threshold and likely test implementation details rather than behavior.

Orphaned Features

smartViews Dexie table retained with no UI surface. share-task-dialog/ and confetti.ts documented as deleted but still shipping. Schema and export/import surface area for unused data.

The Hard Questions

  • If api.vinny.io goes down for 48 hours, what happens? What’s the user experience? What’s the data recovery story? Is there even a backup?
  • The app markets itself as “privacy-first” but stores plaintext tasks on a server with localStorage tokens and no encryption. What is the actual threat model?
  • 19 files and 2,740 lines of sync code for an optional feature on a personal task app — is this proportional to the value it delivers?
  • Are the coverage-boost / gap-closing test files testing behavior or gaming the 80% threshold? If removed, does coverage actually meet the stated standard?
  • MCP server requires cloud sync to function. It cannot access local-only IndexedDB data. Does this limit the “privacy-first” story for AI-powered task management?

Architecture Decision Records

Key architectural decisions are captured in ADRs under docs/adr/. Each documents the context, decision, consequences, and alternatives considered.

ADR-0001

Client-Side Only IndexedDB — All data stored locally. No server required for core functionality.

ADR-0002

PocketBase Cloud Sync — Self-hosted PocketBase for opt-in multi-device sync.

ADR-0003

Last-Write-Wins Conflict Resolution — Whole-record LWW using client_updated_at timestamps.

ADR-0004

PWA Architecture — Progressive Web App with service worker and offline support.

ADR-0005

MCP Server Integration — 20-tool Claude Desktop integration via stdio JSON-RPC.

ADR-0006

Analytics Modular Design — Modular metrics, streaks, tags, and trends system.

ADR-0007

Notification System Design — Modular notification display, permissions, settings, and badges.

ADR-0008

BFS Circular Dependency Detection — Breadth-first search to prevent dependency cycles.

ADR-0009

Smart Views System — Custom filtered views (UI removed in v9, schema retained).

ADR-0010

Agent Discovery Surface — Exposing task data to AI agents via structured tools.

ADR-0011

v9 Single Matrix Refactor — Simplified from ~30 files to single-matrix shell + capture bar.

ADR-0012

SW Multi-Cache Strategy — Three purpose-specific caches reducing post-deploy re-download by ~95%.
